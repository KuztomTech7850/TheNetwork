# Integration — HolistiveHive App

> Repo: [HolistiveHive_App](https://github.com/KuztomTech7850/HolistiveHive_App)

HolistiveHive is a modular personal/operational dashboard. The Network
is already listed in its integration stack as "Community backbone."

## Integration Path

| HolistiveHive Phase | Network Integration |
|---|---|
| Phase 0–1 (current) | No changes — HolistiveHive builds independently |
| Phase 2 (AI Calibration) | Network identity (DID) feeds the AI priority queue; community resource data pulled as pillar feeds |
| Phase 3 (Fork Refinement) | Personal fork uses Network profile capsule as identity/context layer |
| Phase 4 (Public Release) | HolistiveHive ships as a reference implementation of a Network-connected personal app |

## What HolistiveHive Gets From The Network
- Portable DID — one identity across all Network instances
- Community resource feeds — Board records surfaced as pillar modules
- Voting module — governance participation from the dashboard

## What Stays Separate
- Private pillars (health, finances, household) — never sent to The Network
- HolistiveHive's Five Pillar schema — Network-agnostic, configurable per operator

## Status
> Architecture aligned. Implementation begins at HolistiveHive Phase 2.
