# Contributing to The Network

This project is in architectural definition. Contributions at this stage
are research, spec refinement, and open question resolution — not code.

## How to Contribute

1. **Open questions** — See [`docs/open-questions.md`](docs/open-questions.md).
   If you have a well-reasoned answer to any open question, open an issue
   with your proposal and supporting references.

2. **Spec feedback** — Each module spec in [`spec/`](spec/) is a living
   document. Open an issue or PR against any spec file with your suggested
   changes and rationale.

3. **New use cases** — If you have a community use case that The Network
   should support, document it in [`docs/use-cases.md`](docs/use-cases.md)
   and open an issue describing the module requirements.

## Standards

- Every technical claim must have a rationale, reference, or proof of concept.
- Label speculation clearly. Separate hypotheses from verified facts.
- Use inclusive language. This platform is built for vulnerable communities.

## Thread Template

When opening a research or spec issue, use this format:
Title: [Module] — [Topic] (v0.x)
Tags: architecture / governance / research / iteration / coordination / meta

Objective: 1–2 sentences.
Background: How does this fit the larger project?
Open Questions: What needs to be resolved?
References: Links, papers, prior threads.
